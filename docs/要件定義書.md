# 有価証券報告書RAGアプリ 要件定義書

## 1. コンセプト

有価証券報告書を一次情報として企業を深堀りする、**企業分析ツール**。

ワンキャリア・マイナビ・四季報のような二次加工情報ではなく、企業が公式に開示した有価証券報告書を根拠として、財務データの可視化・自然言語での質問を可能にする。

---

## 2. ターゲットユーザー

- 志望企業を絞り込み段階にある就活生
- 有価証券報告書を読みたいが、読み方がわからない・時間がかかる学生
- 投資先を検討する一般人

---

## 3. データソース

| データ | 取得方法 | 更新頻度 |
|---|---|---|
| 有価証券報告書（本文テキスト） | EDINET XBRL ZIPに含まれるiXBRL HTMLをMarkdown変換 | 年1回（取り込み時） |
| 有価証券報告書（財務数値） | EDINET XBRL タグ解析 | 年1回（取り込み時） |
| 最新ニュース | Web検索（Tavily API） | ページ表示時（予定） |

> PDFは使用しない。XBRL ZIPにiXBRL形式の本文が含まれており、PDF変換より高精度なMarkdown変換が可能。

---

## 4. 画面構成

VS Code ライクな IDE レイアウト。企業データをコードエディタのように並べて閲覧できる。

- スプリットペイン対応（2社を並べて比較閲覧）
- ChatPanel は右側に固定・折りたたみ可能・幅リサイズ可能
- Sidebar は折りたたみ可能・幅リサイズ可能

### URLルーティング

```
/          → /[ticker] にリダイレクト（現在は /7203 固定）
/[ticker]  → 企業個別ページ
```

## 5. 機能一覧

### 5.1 企業サマリー表示（事前生成・静的）

有価証券報告書取り込み時にLLMで一度だけ生成してDBに保存。表示時はDBから取得するだけ（LLM不要）。

| セクション | 主なデータ |
|---|---|
| 人的資本 | 従業員数（連結・単体）・平均年齢・平均勤続年数・平均年収・女性管理職比率 + LLMコメント |
| 事業概要 | 事業内容要約・セグメント別売上構成・研究開発費・沿革ハイライト |
| 将来性・経営課題 | 経営方針・対処すべき課題・主要リスク・設備投資状況 |

### 5.2 財務グラフ・KPI（事前生成・静的）

XBRL APIで取得した財務数値をMongoDBに保存してグラフ描画（LLM不要）。

- KPIカード（売上・純利益・ROE・自己資本比率）
- 財務チャート（売上・純利益・ROEの推移）
- 財務テーブル（複数年度の一覧）

### 5.3 RAGチャット（動的）

企業の有価証券報告書に基づいて自然言語で質問に回答する。

- ChatPanel から質問を投稿 → バックエンドの `/api/chat` → MongoDB Vector Search → Claude API
- 質問に関連するセクション名・年度を回答に明示
- 会話履歴はブラウザのローカルストレージに保持（Zustand persist）

### 5.4 最新ニュース取得（予定）

ページ表示時にTavily APIでニュースを自動取得。LLM不使用でコストゼロ。

### 5.5 ニュース×有価証券報告書照合分析（予定）

ボタン操作でニュースと有価証券報告書をLLMで照合し、企業動向を分析。24時間キャッシュ。

### 5.6 ES添削（予定）

ES文章をクエリとしてRAG検索し、有価証券報告書の関連箇所を根拠にLLMがフィードバック。
