# 開発環境

## 前提ツール

| ツール | 用途 | インストール |
|---|---|---|
| uv | Python パッケージ管理 | `curl -LsSf https://astral.sh/uv/install.sh \| sh` |
| Node.js | フロントエンド実行環境 | [nodejs.org](https://nodejs.org) からインストール（LTS版） |
| git | バージョン管理 | 別途インストール |

---

## 1. バックエンドのセットアップ

```bash
cd backend

# 仮想環境の作成・有効化
uv venv                        # .venv/ フォルダを作成
source .venv/bin/activate      # 仮想環境を有効化（プロンプトに (.venv) と表示されれば成功）

# 依存パッケージのインストール
uv pip install -r requirements.txt
```

> **VSCode使用時：** 仮想環境作成後、`Cmd+Shift+P` → `Python: Select Interpreter` → `.venv/bin/python` を選択する。

### パッケージを新たに追加するとき

```bash
uv pip install <パッケージ名>
uv pip freeze > requirements.txt  # requirements.txtに反映
```

### 環境変数の設定

```bash
cp .env.example .env
```

`.env` を編集：

```
MONGODB_URI=mongodb+srv://<user>:<password>@cluster.mongodb.net/recruit_app
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
TAVILY_API_KEY=tvly-...
LANGCHAIN_API_KEY=ls-...       # LangSmith（開発ロギング用）
LANGCHAIN_TRACING_V2=true
```

## 2. フロントエンドのセットアップ

```bash
cd frontend

# 依存パッケージのインストール
npm install
```

### 環境変数の設定

```bash
cp .env.local.example .env.local
```

`.env.local` を編集：

```
NEXT_PUBLIC_API_URL=http://localhost:8000
NEXTAUTH_SECRET=your-random-secret   # openssl rand -base64 32 で生成
NEXTAUTH_URL=http://localhost:3000
```

---

## 3. MongoDB Atlas のセットアップ

1. [MongoDB Atlas](https://cloud.mongodb.com) でアカウント作成
2. 無料クラスター（M0）を作成
3. Database User を作成（ユーザー名・パスワードをメモ）
4. Network Access で `0.0.0.0/0`（全IP許可）を設定
5. 接続文字列を `.env` の `MONGODB_URI` に設定

### Vector Search インデックスの作成

Atlas UI の Search タブから以下のインデックスを作成：

```json
{
  "fields": [
    {
      "type": "vector",
      "path": "embedding",
      "numDimensions": 1536,
      "similarity": "cosine"
    },
    {
      "type": "filter",
      "path": "ticker"
    },
    {
      "type": "filter",
      "path": "type"
    }
  ]
}
```

> `numDimensions: 1536` は OpenAI text-embedding-3-small の次元数

## 4. ローカル起動

ターミナルを2つ開いて起動する。

**ターミナル1：バックエンド**

```bash
cd backend
source .venv/bin/activate
uvicorn api.main:app --reload --port 8000
```

→ http://localhost:8000/docs でSwagger UIが確認できる

**ターミナル2：フロントエンド**

```bash
cd frontend
npm run dev
```

→ http://localhost:3000 でアプリが起動する

## 5. データ取り込み（初回セットアップ）

企業データを取り込むスクリプトを実行する。

```bash
cd backend
source .venv/bin/activate

# 1社取り込み（例：トヨタ = 7203）
python -m ingestion.edinet --ticker 7203 --year 2024

# 実行内容：
#   1. EDINET から PDF・XBRL をダウンロード
#   2. XBRL → 財務数値 → MongoDB financials に保存
#   3. PDF → Markdown → チャンク化 → Embedding → MongoDB chunks に保存
#   4. LLM → サマリー生成 → MongoDB companies に保存
```

## 6. デプロイ

### フロントエンド（Vercel）

1. [Vercel](https://vercel.com) にGitHubリポジトリを連携
2. Root Directory を `frontend` に設定
3. Environment Variables に以下を設定：

```
NEXT_PUBLIC_API_URL=https://your-api.railway.app
NEXTAUTH_SECRET=your-random-secret
NEXTAUTH_URL=https://your-app.vercel.app
```

4. Deploy → 以降は `main` ブランチへのpushで自動デプロイ

### バックエンド（Railway）

1. [Railway](https://railway.app) にGitHubリポジトリを連携
2. Root Directory を `backend` に設定
3. Environment Variables に以下を設定：

```
MONGODB_URI=mongodb+srv://...
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
TAVILY_API_KEY=tvly-...
```

4. Deploy → 以降は `main` ブランチへのpushで自動デプロイ

## 8. コミットメッセージ規約

[Conventional Commits](https://www.conventionalcommits.org/) に準拠する。

```
<type>: <概要（命令形・50字以内）>
```

### type 一覧

| type | 用途 |
|---|---|
| `feat` | 新機能 |
| `fix` | バグ修正 |
| `chore` | ビルド・依存・設定変更（機能に影響しない） |
| `docs` | ドキュメントのみの変更 |
| `refactor` | 動作を変えないコード整理 |
| `test` | テスト追加・修正 |
| `style` | フォーマット等（ロジック変更なし） |

### 具体例

```
feat: EDINET PDFダウンロード機能を追加
feat(rag): サマリーチャンク検索に対応
feat(api): /api/es エンドポイントを追加
fix: Vector Search のフィルター条件が無効になるバグを修正
fix(chat): セッションIDが未設定の場合にエラーが発生する問題を修正
chore: .gitignore にEDINETダウンロードファイルを追加
chore: 依存パッケージを更新
docs: RAG設計書にチャンク戦略を追記
refactor: retriever.py の検索ロジックを関数に切り出し
test: financial Tool の単体テストを追加
```

### ルール

- 概要は日本語OK
- 概要は命令形（「追加した」ではなく「追加」）
- スコープは任意。変更範囲が明確な場合は `feat(rag):` のように付ける

## 8. トラブルシューティング

### MongoDB に接続できない

- Atlas の Network Access に自分のIPが許可されているか確認
- `MONGODB_URI` の形式が正しいか確認（`@` の前後にユーザー名・パスワード）

### Vector Search が動かない

- Atlas のインデックスが `Active` 状態になっているか確認（作成直後は数分かかる）
- `numDimensions` が `1536` になっているか確認

### FastAPI が起動しない

- 仮想環境が有効化されているか確認（`source .venv/bin/activate`）
- `.env` が存在するか確認
