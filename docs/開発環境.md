# 開発環境

## 前提ツール

| ツール | 用途 |
|---|---|
| uv | Python パッケージ管理 |
| Node.js | フロントエンド実行環境 |
| Docker Desktop | コンテナ実行環境 | 
| Infisical CLI | シークレット管理 |
| git | バージョン管理 |

---

## 1. シークレット管理（Infisical）

APIキーやDB接続情報は Infisical で管理する。

### 初回セットアップ

**1. Infisical にログイン**

```bash
infisical login
```

**2. Machine Identity の認証情報を設定**

管理者から `INFISICAL_CLIENT_ID` と `INFISICAL_CLIENT_SECRET` を受け取り、`~/.zshrc` に追加：

```bash
export INFISICAL_CLIENT_ID="your-client-id"
export INFISICAL_CLIENT_SECRET="your-client-secret"
```

```bash
source ~/.zshrc
```

---

## 2. ローカル起動

### Docker を使う場合（推奨）

ルートディレクトリで1コマンドで起動。

```bash
# 初回（イメージビルド＋起動）
infisical run --env=dev --path=/ --recursive -- docker compose up --build

# 2回目以降
infisical run --env=dev --path=/ --recursive -- docker compose up
```

| URL | 内容 |
|---|---|
| http://localhost:3000 | フロントエンド |
| http://localhost:8000/docs | バックエンド Swagger UI |

```bash
# 停止
docker compose down
```

### 手動で起動する場合

ターミナルを2つ開いて起動する。

**ターミナル1：バックエンド**

```bash
cd backend
uv venv && source .venv/bin/activate
uv pip install -r requirements.txt

infisical run --env=dev --path=/ --recursive -- uvicorn api.main:app --reload --port 8000
```

**ターミナル2：フロントエンド**

```bash
cd frontend
npm install
npm run dev
```

## 3. データ取り込み（企業を追加するとき）

```bash
cd backend
source .venv/bin/activate

infisical run --env=dev --path=/ --recursive -- python -m ingestion.ingest --ticker 7203 --year 2025 --summarizer claude
```

1コマンドで以下をすべて実行する：

1. EDINET API から XBRL ZIP・PDF をダウンロード（`backend/data/{会社名}/{year}/` に保存）
2. iXBRL HTML → Markdown に変換（html_parser.py）
3. 500字・100字overlapでチャンク化（chunker.py）
4. Voyage AI でベクトル化 → MongoDB `chunks` に保存（embedding.py）
5. XBRL から財務数値を抽出 → MongoDB `financials` に保存（xbrl_parser.py）
6. Claude API で企業サマリーを生成 → MongoDB `companies` に保存（summarizer.py）

## 4. デプロイ（Cloudflare Workers）

フロントエンドは `@opennextjs/cloudflare` で Cloudflare Workers にデプロイする。GitHub連携で自動デプロイが設定済み。

| ブランチ | 実行コマンド | 動作 |
|---------|------------|------|
| `main` | `npm run deploy` | 本番にデプロイ |
| その他 | `npm run upload` | プレビューバージョンをアップロード |

本番URL: `https://corplenz.<アカウントサブドメイン>.workers.dev`

プレビューURL形式: `https://<ブランチ名>-corplenz.<アカウントサブドメイン>.workers.dev`

### 手動デプロイ

```bash
cd frontend
npm run deploy
```

### Cloudflare Dashboard の設定箇所

- **Build Variables**（Workers > corplenz > Build > Variables）: `NEXT_PUBLIC_*` の登録
- **CORS**（Railwayの環境変数）: `ALLOWED_ORIGINS`（本番URL）と `ALLOWED_ORIGIN_REGEX`（プレビューURL用正規表現）
