# 開発環境

## 前提ツール

| ツール | 用途 | インストール |
|---|---|---|
| uv | Python パッケージ管理 | `curl -LsSf https://astral.sh/uv/install.sh \| sh` |
| Node.js | フロントエンド実行環境 | [nodejs.org](https://nodejs.org)（LTS版） |
| Docker Desktop | コンテナ実行環境 | [docker.com](https://www.docker.com/products/docker-desktop/) |
| Infisical CLI | シークレット管理 | `brew install infisical/get-cli/infisical` |
| git | バージョン管理 | 別途インストール |

---

## 1. シークレット管理（Infisical）

APIキーやDB接続情報は Infisical で管理する。`.env` ファイルは使わない。

### 初回セットアップ

**1. Infisical にログイン**

```bash
infisical login
```

**2. Machine Identity の認証情報を設定**

管理者から `INFISICAL_CLIENT_ID` と `INFISICAL_CLIENT_SECRET` を受け取り、`~/.zshrc` に追加：

```bash
export INFISICAL_CLIENT_ID="your-client-id"
export INFISICAL_CLIENT_SECRET="your-client-secret"
```

```bash
source ~/.zshrc
```

**3. 動作確認**

```bash
infisical run --env=dev -- echo "OK"
# "Injecting X Infisical secrets into your application process" と表示されれば成功
```

## 2. ローカル起動

### Docker を使う場合（推奨）

ルートディレクトリで1コマンドで起動。

```bash
# 初回（イメージビルド＋起動）
infisical run --env=dev -- docker compose up --build

# 2回目以降
infisical run --env=dev -- docker compose up
```

| URL | 内容 |
|---|---|
| http://localhost:3000 | フロントエンド |
| http://localhost:8000/docs | バックエンド Swagger UI |

```bash
# 停止
docker compose down

# ログを見る
docker compose logs -f          # 全サービス
docker compose logs -f backend  # バックエンドのみ
```

ソースコードの変更は自動反映される（ホットリロード）。

### 手動で起動する場合

ターミナルを2つ開いて起動する。

**ターミナル1：バックエンド**

```bash
cd backend
uv venv && source .venv/bin/activate
uv pip install -r requirements.txt

infisical run --env=dev -- uvicorn api.main:app --reload --port 8000
```

**ターミナル2：フロントエンド**

```bash
cd frontend
npm install
npm run dev
```

## 3. データ取り込み（初回・企業を追加するとき）

### ステップ1：EDINET から XBRL ZIP をダウンロード

```bash
cd backend
source .venv/bin/activate

infisical run --env=dev -- python -m ingestion.edinet --ticker 7203 --year 2024
```

成功すると `backend/data/{会社名}/{year}/{docID}_xbrl.zip` に保存される。

### ステップ2：XBRL ZIP を取り込み → MongoDB に保存

```bash
infisical run --env=dev -- python -m ingestion.ingest \
  --zip backend/data/トヨタ自動車株式会社/2025/S100VWVY_xbrl.zip \
  --ticker 7203 \
  --year 2024
```

実行内容：
1. iXBRL HTML → Markdown に変換（html_parser.py）
2. 500字・100字overlapでチャンク化（chunker.py）
3. Voyage AI voyage-finance-2 でベクトル化（embedding.py）
4. MongoDB の `chunks` コレクションに保存

## 5. MongoDB Atlas のセットアップ（初回のみ）

1. [MongoDB Atlas](https://cloud.mongodb.com) でクラスター作成（M0 無料枠）
2. Database User を作成（ユーザー名・パスワードを Infisical の `DB_USER` / `DB_PASSWORD` に登録）
3. Network Access で `0.0.0.0/0` を許可
4. Vector Search インデックスを `chunks` コレクションに作成（**取り込み前に実施不要**）

> Vector Search インデックスは `ingest.py` を初回実行したときに自動作成される（`retriever.py` の `create_vector_index()` で作成・待機）。

手動で作成する場合は Atlas UI の Search タブから：

```json
{
  "fields": [
    { "type": "vector", "path": "embedding", "numDimensions": 1024, "similarity": "dotProduct" },
    { "type": "filter", "path": "ticker" }
  ]
}
```

## 7. コミットメッセージ規約

[Conventional Commits](https://www.conventionalcommits.org/) に準拠。

```
<type>: <概要（命令形・50字以内）>
```

| type | 用途 |
|---|---|
| `feat` | 新機能 |
| `fix` | バグ修正 |
| `chore` | ビルド・依存・設定変更（機能に影響しない） |
| `docs` | ドキュメントのみの変更 |
| `refactor` | 動作を変えないコード整理 |
| `test` | テスト追加・修正 |
| `style` | フォーマット等（ロジック変更なし） |

## 8. トラブルシューティング

### MongoDB に接続できない

- Atlas の Network Access に自分のIPが許可されているか確認
- Infisical の `DB_USER` / `DB_PASSWORD` が正しく設定されているか確認

### Vector Search が動かない

- Atlas のインデックスが `Active` 状態か確認（作成直後は数分かかる）
- `numDimensions: 1024`、`similarity: dotProduct` になっているか確認

### FastAPI が起動しない

- 仮想環境が有効化されているか確認（`source .venv/bin/activate`）
- `infisical run --env=dev --` が先頭についているか確認

### Docker コンテナが起動しない

- Docker Desktop が起動しているか確認
- ポート競合を確認（3000・8000番）
- イメージを再ビルド：`infisical run --env=dev -- docker compose up --build`

### Infisical のシークレットが注入されない

- `echo $INFISICAL_CLIENT_ID` で環境変数が設定されているか確認
- 未設定なら `source ~/.zshrc` を実行
