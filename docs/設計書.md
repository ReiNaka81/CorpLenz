## 3. データ取り込みパイプライン

```
① EDINET API
   └── 書類一覧取得 → docID 取得
          ├── PDF ダウンロード
          │     └── pymupdf4llm → Markdown
          │             └── セクション分割 → チャンク化（500字・100字overlap）
          │                     ├── OpenAI Embedding → chunks コレクションへ保存
          │                     └── LLM → サマリーチャンク生成 → chunks へ保存
          └── XBRL ダウンロード
                └── xbrl_parser → Pydantic モデルへ変換 → financials コレクションへ保存

② 企業サマリー生成（LLM・1回だけ）
   └── chunks から主要テキスト取得 → LLM → companies.summaries へ保存
```

### スクリプト実行例

```bash
python -m backend.ingestion.edinet --ticker 7203 --year 2024
```

## 4. RAGパイプライン設計

### 4.1 質問ルーティング（router.py）

```
ユーザー質問
    │
    ▼
router.py（LLMで分類）
    ├── 数値系（売上・利益率・PER等）　　→ financial Tool
    ├── サマリー系（概要・まとめて等）　　→ サマリーチャンク検索 → LLM
    ├── テキスト系（戦略・リスク・ES等）　→ 通常RAG検索 → LLM
    └── 横断系（競合比較等）　　　　　　　→ compare Tool + RAG
```

### 4.2 RAG検索（retriever.py）

```python
def retrieve(query: str, ticker: str, chunk_type: str = "chunk", top_k: int = 5):
    embedding = openai.embed(query)
    return db.chunks.aggregate([
        {
            "$vectorSearch": {
                "index": "vector_index",
                "queryVector": embedding,
                "path": "embedding",
                "numCandidates": 100,
                "limit": top_k,
                "filter": {"ticker": ticker, "type": chunk_type}
            }
        }
    ])
```

### 4.3 横断検索（複数社対応）

```python
def retrieve_multi(query: str, tickers: list[str], top_k: int = 5):
    # 質問文から言及された会社のtickerをLLMで抽出し、検索対象を動的に拡張
    embedding = openai.embed(query)
    return db.chunks.aggregate([
        {
            "$vectorSearch": { ..., "filter": {"ticker": {"$in": tickers}} }
        }
    ])
```

## 5. LangChain Tool 設計

### 5.1 financial Tool

```python
@tool
def get_financial_data(ticker: str, metrics: list[str], year: int) -> str:
    """
    企業の財務数値を取得・計算する。
    売上・営業利益・純利益・ROE・営業利益率・PER等に使う。
    """
    data = db.financials.find_one({"ticker": ticker, "year": year})
    result = {}
    if "営業利益率" in metrics:
        result["営業利益率"] = data["operating_profit"] / data["revenue"]
    # ... 他の指標も同様
    return json.dumps(result, ensure_ascii=False)
```

### 5.2 news Tool

```python
@tool
def search_news(company_name: str) -> str:
    """企業の最新ニュースをWeb検索で取得する。"""
    results = tavily.search(f"{company_name} 最新ニュース")
    return format_news(results)
```

### 5.3 compare Tool

```python
@tool
def compare_companies(tickers: list[str], aspect: str) -> str:
    """
    複数企業を比較する。競合比較・業界比較に使う。
    aspect: "財務" | "事業" | "リスク"
    """
    chunks = retrieve_multi(aspect, tickers)
    return format_comparison(chunks)
```

## 7. チャットフロー全体図

```
[ユーザー] 質問入力
    │
    ▼
[FastAPI /api/chat]
    │
    ▼
[router.py] 質問を分類
    │
    ├─── 数値系 ──────────────▶ [financial Tool] ──▶ MongoDB financials
    │
    ├─── サマリー系 ───────────▶ [retriever.py]  ──▶ MongoDB chunks (summary)
    │                                                        │
    ├─── テキスト系 ───────────▶ [retriever.py]  ──▶ MongoDB chunks (chunk)
    │                                                        │
    └─── 横断系 ────────────── ▶ [compare Tool]  ──▶ MongoDB chunks (multi ticker)
                                                            │
                                                            ▼
                                                    [Claude API]
                                                            │
                                                            ▼
                                                    [ストリーミングレスポンス]
```

## 8. ES添削フロー

```
[ユーザー] ES文章を貼り付け
    │
    ▼
[FastAPI /api/es]
    │
    ├─▶ ESをそのままクエリに → [retriever.py] → 有報の関連チャンク取得
    │
    └─▶ [Claude API]
            入力：ES文章 + 有報チャンク
            指示：有報の情報を根拠にESを添削・フィードバックせよ
            │
            ▼
        フィードバック + 根拠となった有報セクションを返す
```
