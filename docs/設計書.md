## 2. MongoDB コレクション設計

### 2.1 `chunks`（RAGチャンク・実装済み）

有価証券報告書本文をチャンク化・ベクトル化して格納。RAGチャットの検索対象。

```json
{
  "_id": "ObjectId",
  "ticker": "7203",
  "year": 2025,
  "section": "事業の状況",
  "text": "...",
  "embedding": [0.012, -0.034, ...]
}
```

Vector Search インデックス：

```json
{
  "fields": [
    { "type": "vector", "path": "embedding", "numDimensions": 1024, "similarity": "dotProduct" },
    { "type": "filter", "path": "ticker" }
  ]
}
```

> `numDimensions: 1024` は Voyage AI voyage-finance-2 の次元数。similarity は `dotProduct`。

### 2.2 `financials`（財務数値・実装済み）

XBRLから抽出した財務数値を格納。**単位は億円**（円から変換済み）。`ticker` + `year` をキーにupsert。

```json
{
  "ticker": "7203",
  "year": 2024,
  "accounting_standard": "IFRS",
  "revenue": 452351,
  "pretax_profit": 71570,
  "net_profit": 53584,
  "total_assets": 952174,
  "equity": 332190
}
```

> `operating_profit` は廃止。IFRSは営業利益の開示を義務付けていないため、IFRS/J-GAAP共通の `pretax_profit` を使用。

### 2.3 `companies`（企業サマリー・実装済み）

取り込み時にLLMで生成した構造化サマリーを格納。ページ表示時はDBから取得するだけでLLM不要。`ticker` をキーにupsert。

- `name` は EDINET APIの `filerName` から自動取得
- `sector` は `edinetCode` をキーに `data/sector_map.csv`（東証33業種）から自動変換

```json
{
  "ticker": "7203",
  "name": "トヨタ自動車株式会社",
  "sector": "輸送用機器",
  "summary": {
    "human_capital": {
      "employee_count_consolidated": 383853,
      "employee_count_standalone": 71515,
      "average_age": 40.7,
      "average_tenure": 15.6,
      "average_salary": 982,
      "female_manager_ratio": 4.0,
      "commentary": "..."
    },
    "business": {
      "description": "...",
      "segments": [{ "name": "自動車", "revenue_ratio": 0.89 }],
      "history_highlights": "..."
    },
    "management": {
      "policy": "...",
      "challenges": "...",
      "risks": "..."
    }
  },
  "updated_at": "2025-04-24T00:00:00Z"
}
```

### 2.4 `analysis_cache`（照合分析キャッシュ・予定）

```json
{
  "_id": "7203",
  "result": "...",
  "cached_at": "2024-10-01T10:00:00",
  "ttl": 86400
}
```

## 3. データ取り込みパイプライン（実装済み）

```
① EDINET API（edinet.py）
   └── 書類一覧取得 → docID 取得
         └── XBRL ZIP ダウンロード → backend/data/{会社名}/{year}/ に保存
         └── filerName（会社名）・docID を返す ← ingest.py に渡す

② ingest.py（エントリーポイント）
   └── XBRL ZIP を受け取り
         ├── html_parser.py  → iXBRL HTML → Markdown（セクションごと）
         ├── chunker.py      → 500字・100字overlapでチャンク化
         ├── embedding.py    → Voyage AI voyage-finance-2 でベクトル化 → MongoDB chunks に保存
         ├── xbrl_parser.py  → XBRLタグ → 財務数値抽出 → MongoDB financials に保存（億円）
         └── summarizer.py   → Claude API で構造化サマリー生成 → MongoDB companies に保存
```

### 実行コマンド

```bash
# 1コマンドでダウンロード〜MongoDB保存まで完結
python -m ingestion.ingest --ticker 7203 --year 2025
```

実行内容：

1. EDINET API から XBRL ZIP・PDF をダウンロード（`backend/data/{会社名}/{year}/` に保存）
2. iXBRL HTML → Markdown に変換（html_parser.py）
3. 500字・100字overlapでチャンク化（chunker.py）
4. Voyage AI でベクトル化 → MongoDB `chunks` に保存（embedding.py）
5. XBRL から財務数値を抽出 → MongoDB `financials` に保存（xbrl_parser.py）
6. Claude API で企業サマリーを生成 → MongoDB `companies` に保存（summarizer.py）

## 4. RAGパイプライン（実装済み）

```
[ユーザー] 質問入力（ChatPanel）
    │
    ▼
[POST /api/chat]  { query, ticker }
    │
    ▼
[retriever.py]
    └── query を Voyage AI でベクトル化
    └── MongoDB Vector Search（ticker でフィルタ・top_k=5）
    └── チャンク（text, section, year）を返す
    │
    ▼
[chain.py]
    └── Claude API（claude-sonnet）に質問 + チャンクを渡す
    └── 回答にセクション名・年度を明示させる
    │
    ▼
[レスポンス] { answer: "..." }
```

### retriever.py の Vector Search クエリ

```python
pipeline = [
    {
        "$vectorSearch": {
            "index": "vector_index",
            "queryVector": get_embedding(query, input_type="query"),
            "path": "embedding",
            "numCandidates": 100,
            "limit": top_k,
            "filter": {"ticker": ticker},
        }
    },
    {"$project": {"_id": 0, "text": 1, "section": 1, "year": 1}},
]
```

### chain.py のシステムプロンプト方針

- 参考資料（チャンク）のみを根拠として回答させる
- 使用箇所のセクション名・年度を明示させる（例：`[経営成績 2023] によると〜`）
- 資料から回答できない場合は固定メッセージで返す

## 5. APIエンドポイント

### 実装済み

#### POST `/api/chat`

```
Request:  { "query": "営業利益率は？", "ticker": "7203" }
Response: { "answer": "..." }
```

#### GET `/api/company?ticker=7203`

企業サマリーと財務データをDBから返す。取り込み時に生成済みのデータをそのまま返すためLLM呼び出しなし。

Response: {
  "name": "トヨタ自動車株式会社",
  "sector": "輸送用機器",
  "summary": {
    "human_capital": { "employee_count_consolidated": 383853, ... },

  "financials": [
    { "year": 2021, "revenue": 279299, "net_profit": 28501, ... },
    ...
  ]
}
```

#### GET `/api/companies`

DBに登録済みの全企業の一覧を返す。サイドバーの企業リスト表示に使用。

```
Response: {
  "companies": [
    { "ticker": "7203", "name": "トヨタ自動車株式会社", "sector": "輸送用機器" },
    ...
  ]
}
```

### 予定

#### GET `/api/news?ticker=7203`

Tavily APIでニュースを取得して返す。

```
Response: { "articles": [{ "title": "...", "summary": "...", "url": "...", "published_at": "..." }] }
```

#### POST `/api/analyze`

ニュースと有価証券報告書チャンクをLLMで照合分析。24時間キャッシュ。

```
Request:  { "ticker": "7203" }

ES文章をクエリとしてRAG検索し、有価証券報告書の根拠でLLMが添削。

```
Request:  { "ticker": "7203", "es_text": "御社の..." }
Response: { "feedback": "...", "sources": [...] }
```

## 6. フロントエンド状態管理（Zustand）

`appStore.ts` で以下を一元管理：

| 状態 | 内容 |
|---|---|
| `companies` | DBから取得した企業一覧（AppLayout マウント時に `/api/companies` でfetch） |
| `selectedCompany` | 左ペインで選択中の企業 |
| `openTabs` | 左ペインの開いているタブ |
| `splitEnabled` | スプリットペインのON/OFF |
| `rightPaneCompany` | 右ペインで選択中の企業 |
| `rightPaneTabs` | 右ペインの開いているタブ |
| `chatHistories` | 企業ごとのチャット履歴（persist対象） |
| `chatOpen` | ChatPanelの開閉状態 |
| `searchQuery` / `activeFilter` | サイドバーの検索・フィルタ状態 |

チャット履歴のみ `localStorage`（`corplens-storage` キー）に永続化。それ以外はリロードでリセット。

`Company.color` は DBに保存せず、`tickerToColor(ticker)` で ticker から決定論的に生成する。
